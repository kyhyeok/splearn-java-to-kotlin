# ADR-0009: Flyway 마이그레이션 불변 원칙

## 상태

승인됨

## 컨텍스트

Spring Data JDBC 전환 후 스키마 관리를 Flyway로 이관했다. 이미 적용된 마이그레이션 파일을 수정하면 Flyway checksum 불일치로 애플리케이션이 기동되지 않으며, 운영 환경과 개발 환경 간 스키마가 불일치할 위험이 있다.

## 결정

**적용된 Flyway 마이그레이션 파일(`V*.sql`)은 절대 수정·삭제하지 않는다.**

### 규칙

1. 한 번 커밋된 `V*.sql`은 내용을 바꾸지 않는다 (공백·주석 포함).
2. 스키마 변경이 필요하면 **새 버전 파일**을 추가한다 (`V3__...sql`, `V4__...sql`).
3. 개발 중 아직 적용하지 않은 파일은 수정 가능하다 — 단, 다른 팀원에게 공유된 이후는 불변.
4. 실수로 잘못된 마이그레이션을 적용했다면 롤백 SQL을 새 버전으로 추가한다.

### H2 테스트 환경

- `spring.datasource.url`에 `MODE=MySQL;DATABASE_TO_LOWER=TRUE` 옵션을 사용한다.
- MySQL 전용 DDL(`ENGINE=InnoDB` 등)은 H2와 호환되지 않으므로 쓰지 않는다.

## 결과

- 모든 환경에서 동일한 스키마 이력 보장.
- Flyway 체크섬 오류 방지.
- 스키마 변경 이력이 마이그레이션 파일로 추적 가능.
